# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public GitHub issues.

Use GitHub private vulnerability reporting if it is enabled for the repository.
If that is unavailable, email security@pwrdrvr.com with:

- A concise description of the issue.
- Steps to reproduce or a proof of concept.
- Affected versions or commits, if known.
- Any logs or screenshots that help validate impact.

Do not include secrets, tokens, private user data, or exploit payloads beyond
what is necessary to demonstrate the issue.

## Response Expectations

PwrDrvr LLC will acknowledge valid reports as soon as practical, triage impact,
and coordinate a fix and disclosure timeline with the reporter.
